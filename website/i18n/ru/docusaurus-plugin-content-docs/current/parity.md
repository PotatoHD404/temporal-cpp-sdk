# Возможности и паритет

**Данный SDK не находится в паритете с официальными Temporal SDK и в ближайшее время им не станет.** Официальные SDK для Go/Java/Python/TS/.NET — это годы работы целых команд; один только движок Temporal на Go занимает десятки тысяч строк, а полная поверхность API (расписания (schedules), Nexus, версионирование, фреймворк для тестирования, безопасность, наблюдаемость, operator/cloud-клиент…) огромна.

Что представляет собой этот проект: подлинно работающее, **полностью протестированное ядро** модели программирования Temporal — достаточное для написания и запуска реальных воркфлоу, которые оркестрируют активности, реагируют на сигналы/запросы (query)/обновления (update), компонуют дочерние воркфлоу и селекторы, а также выполняют continue-as-new — на нативном движке C++ со sticky-кэшем. Эта страница — честная картина текущего состояния.

**Условные обозначения:** ✅ реализовано и протестировано · 🟡 частично · ❌ не реализовано

## Дорожная карта {#roadmap}

Приблизительный порядок приоритетов (подробнее — в `docs/ROADMAP.md` репозитория):

1. **Упрочение детерминизма** — ✅ обнаружение недетерминизма (команды сверяются с историей при реплее); осталось: пагинация истории, ограниченный LRU sticky-кэш, троттлинг heartbeat и обнаружение отмены.
2. **Поверхность возможностей воркфлоу** — валидаторы обновлений, расширенные области отмены, случаи channel для селекторов, SideEffect/MutableSideEffect, версионирование `GetVersion`, локальные активности.
3. **Производственные требования** — TLS/mTLS + аутентификация по API-ключу, интерсепторы, метрики и трейсинг, proto/protoJSON-конвертеры + payload-кодеки, настройка воркеров.
4. **Расширение охвата** — расписания (schedules), Nexus, версионирование воркеров, фреймворк для реплея/тестирования, search attributes, расширенная поверхность клиента.

Если нужная вам возможность находится в столбце ❌ — её действительно ещё нет. Пожалуйста, не делайте противоположных выводов, опираясь на работающее ядро.
